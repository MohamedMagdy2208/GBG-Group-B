5) Edge-case and ambiguity questions

These are important for benchmarking chatbot quality, because many systems fail here.

Show the best customer.
Who is the top employee?
Which is the most popular album?
What is the biggest genre?
Show sales by location.
Which customers are active?
What is the best-selling music?
Which artist performs best?
What is the average purchase?
Which country is best?

These are intentionally vague. A strong chatbot should:

ask for clarification, or
make a clear assumption and state it

Example:
“best customer” could mean:

highest total revenue
most invoices
most tracks purchased
6) Error-handling benchmark questions

These test whether the chatbot safely handles impossible or unsupported requests.

Show revenue by quarter for 2015.
Which customers subscribed last month?
What is the refund rate?
Which products were returned most often?
Show profit margin by genre.
Which customers used coupons?

These fields do not exist in standard Chinook. A good chatbot should say the data is unavailable instead of hallucinating.

8) Conversation-style benchmark questions

These test multi-turn memory and context.

Who are the top 5 customers by total spending?
Of those, which ones are from Canada?
For those customers, what genres do they buy most?
Which of those genres brings in the most revenue overall?
Show that by year.

A good chatbot should preserve context from “those”.

9) Recommended scoring criteria

For each question, score the chatbot on:

Schema understanding: did it use the right tables and columns?
SQL correctness: does the query run and return the expected result?
Reasoning: did it interpret the request correctly?
Robustness: did it avoid hallucinating missing fields?
Clarity: did it explain assumptions when needed?
Conversational memory: did it handle follow-up references correctly?

A simple 0–2 scale works well:

0 = wrong / hallucinated
1 = partially correct
2 = fully correct
